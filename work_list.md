## 1. 개요

### 목적

> 컨펌된 온, 오프라인 RX 주문만 모아 작업 상태를 관리할 수 있다.


### 적용 범위

> New
  

### 주요 용어

| 용어 | 설명 |
| --- | --- |
| RX | 고객 처방 정보에 맞춰진 커스텀 렌즈 |
| C.O.F | Lens Only 타입 렌즈 |
| 작업 라벨 | 작업 건 별 구분을 위해 출력하는 라벨 |
| 송장 | 배송을 위해 출고 시, 목적지 정보가 담긴 필수 라벨 |


### 공통 사항

> 계정 권한 그룹의 메뉴 조회 조건이 있어야, 메뉴 노출이 된다.
> 필터 드롭다운 데이터 노출 기준은 계정에 설정된 브랜드 - 법인 - 스토어에 따라 조회된다.

## 2. 검색 영역

### UI 형태

- 드롭다운(Dropdown) 형태

### 검색 필드 구성

| 순서 | 필드명 | 선택 유형       | 기본값 | 비고 |
| ---- | ------ | --------------- | ------ | ---- |
| 1    | BP | Single Select | 전체 |  권한 조건 |
| 2    | Store | Multiple Select | 전체 | 권한 조건 |
| 2    | Channel     | Single Select | 전체 |      |

### 동작 정책

- **기본값**: 모든 검색 유형은 `전체`를 기본값으로 고정한다.
- **필드 활성화**: BP 설정에 따라 Store 드롭다운 필드가 활성화된다.
- **다중 선택**: 스토어는 체크박스 드롭다운을 통해 복수 선택 검색이 가능하다.

- 기간(Date)

### 검색 필드 구성

| 순서 | 필드명 | 선택 유형       | 기본값 | 비고 |
| ---- | ------ | --------------- | ------ | ---- |
| 1    | Date Type | Single Select | Order Date | Approval Date, Complete Date |
| 2    | Start(시작일)| Calendar Picker | Today-30D |  |
| 2    | End(종료일) | Calendar Picker | Today |  |


### 동작 정책

- **기본값**: `Order Date`를 기본값으로 고정한다.
- **기본값**: 조회하는 시점의 'today`를 기준으로 -30D 데이터 조회를 기본값으로 고정한다.

## 3. 리스트 탭

> 작업 후 ➊ 고객에게로 배송 또는 ➋ 스토어 픽업 유형에 따라 리스트를 구분한다. (출고 방식 상이)

### 탭 구성

| 탭  | 탭명      | 설명 |
| --- | --------- | ---- |
| 1   | To Customer | 온라인 주문 + 오프라인 / 고객 배송 유형 |
| 2   | To Store | 오프라인 / 스토어 배송 유형 |

### 탭 전환 정책

- 메뉴 이동 및 화면 리프레시를 제외하고, **탭 간 이동 시 페이지네이션이 유지**되어야 한다.
- 페이지네이션 고정은, **페이지 번호(위치) 및 row 수 상태** 유지를 의미한다.
- **검색 조건이 반영된 상태**로 탭 전환이 된다. 
- 해당 정책은 탭이 구성된 **모든 메뉴에 공통 적용**되는 기본값이다.

## 4. 리스트 총 건수 및 엑셀 다운로드

### 총 건수

- 표시 형식: `총 000 건`

### 엑셀 다운로드

| 항목           | 정책                              |
| -------------- | --------------------------------- |
| 다운로드 범위  | 현재 필터 기준 전체 데이터        |
| 페이징 처리    | 페이징 제외, 전체 데이터 포함     |
| 빈 데이터 처리 | 없는 데이터는 빈값(`""`)으로 추출 |
| 최대 다운로드  | **최대 3,000 row**                |
